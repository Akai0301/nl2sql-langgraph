# 项目架构说明

> 更新时间：2026-04-01

## 项目概述

**项目名称**：nl2sql-langgraph
**项目类型**：LangGraph 梭子形 NL2SQL 流水线
**技术栈**：FastAPI + psycopg + LangGraph + Vue 3

nl2sql-langgraph 是一个基于 LangGraph 的自然语言转 SQL 系统。用户可以使用自然语言提问，系统自动生成 SQL 并执行查询，返回数据结果。

---

## 架构设计

### 梭子形架构

本项目采用 LangGraph 的梭子形（Fan-out/Fan-in）架构：

```
问题 → 分析器 → [并行检索] → 合并 → 元数据分析 → SQL生成 → SQL执行
                      ↓
          ┌───────────┼───────────┐
          ↓           ↓           ↓
     知识检索     指标检索     元数据检索
          └───────────┼───────────┘
                      ↓
                上下文合并
```

### 架构特点

| 特点 | 说明 |
|------|------|
| **扇出/扇入** | 多个检索节点并行执行，合并节点等待所有检索完成 |
| **StateGraph** | 使用 LangGraph 的状态图编排流程 |
| **TypedDict State** | 类型安全的状态定义 |
| **SSE 流式** | 实时返回执行进度 |

## 执行流程

### 完整流程

1. **问题分析**：提取关键词和同义词
2. **并行检索**：
   - 知识检索：查询 `enterprise_kb`
   - 指标检索：查询 `metrics_catalog`
   - 元数据检索：查询 `lake_table_metadata`
3. **上下文合并**：提取候选表
4. **元数据分析**：选择表和 Join 逻辑
5. **SQL 生成**：规则模式或 LLM 模式
6. **SQL 执行**：执行并返回结果

### SSE 事件流

```
event: init         → 初始化图结构
event: node_start   → 节点开始执行
event: node_complete → 节点执行完成
event: result       → 最终结果
```

| 变量名 | 说明 |

| 变量名 | 说明 | 默认值 |

## 扩展指南

### 添加新节点

1. 在 `app/state.py` 添加状态字段
2. 在 `app/nodes.py` 实现节点函数
3. 在 `app/graph_builder.py` 添加节点和边

### 添加新检索源

1. 在 `app/tools.py` 添加检索函数
2. 在 `app/nodes.py` 添加节点
3. 更新 `app/graph_builder.py` 并行边

### 自定义 SQL 生成

1. 修改 `app/prompt_templates.py`
2. 或修改 `app/nodes.py` 的规则生成逻辑
