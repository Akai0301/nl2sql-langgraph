# 环境配置指南

> **版本**：v1.0.0
> **最后更新**：2026-04-01
> **适用项目**：nl2sql-langgraph

---

## 概述

nl2sql-langgraph 采用双数据库架构，需要配置 PostgreSQL 和 MySQL 两个数据库连接。本文档说明所有环境变量的配置方法。

---

## 环境变量清单

### PostgreSQL 配置（问数业务数据）

| 变量名 | 说明 | 示例值 | 必填 |
|--------|------|--------|------|
| `POSTGRES_DSN` | PostgreSQL 连接串 | `postgresql://postgres:postgres@localhost:5432/nl2sql` | ✅ |

**DSN 格式**：
```
postgresql://{用户名}:{密码}@{主机}:{端口}/{数据库名}
```

### MySQL 配置（平台数据）

| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| `MYSQL_HOST` | MySQL 主机 | `localhost` | ❌ |
| `MYSQL_PORT` | MySQL 端口 | `3306` | ❌ |
| `MYSQL_USER` | MySQL 用户名 | `root` | ❌ |
| `MYSQL_PASSWORD` | MySQL 密码 | - | ❌ |
| `MYSQL_DATABASE` | MySQL 数据库名 | `nl2sql_platform` | ❌ |

### LLM 配置

| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| `USE_MOCK_LLM` | 是否使用规则模式 | `true` | ❌ |
| `OPENAI_API_KEY` | OpenAI API Key | - | LLM 模式必填 |
| `OPENAI_API_BASE` | API 代理地址 | - | ❌ |
| `OPENAI_MODEL` | 模型名称 | `gpt-4o-mini` | ❌ |

### SQL 执行配置

| 变量名 | 说明 | 默认值 | 必填 |
|--------|------|--------|------|
| `SQL_MAX_ROWS` | 返回最大行数 | `200` | ❌ |
| `SQL_MAX_ATTEMPTS` | SQL 执行失败重试次数 | `2` | ❌ |

## 配置步骤

### 1. 创建配置文件

```powershell
# Windows PowerShell
copy .env.example .env

# Linux/Mac
cp .env.example .env
```

### 2. 编辑配置文件

```env
# PostgreSQL（问数业务数据）
POSTGRES_DSN=postgresql://postgres:your_password@localhost:5432/nl2sql

# MySQL（平台数据）
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=nl2sql_platform

# LLM 配置
USE_MOCK_LLM=false
OPENAI_API_KEY=sk-xxx
OPENAI_API_BASE=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini

# SQL 执行配置
SQL_MAX_ROWS=200
SQL_MAX_ATTEMPTS=2
```

### 3. 验证配置

```python
# 验证 PostgreSQL 连接
from app.tools import get_postgres_dsn, execute_sql
print(get_postgres_dsn())

# 验证 MySQL 连接
from app.mysql_tools import get_mysql_connection
conn = get_mysql_connection()
print(conn)
conn.close()
```

## 运行模式

### Mock 模式（默认）

`USE_MOCK_LLM=true` 时，系统使用规则引擎生成 SQL，无需 LLM API：

- 优点：无需 API Key，响应快速
- 缺点：只能处理预定义的问题模板

**适用场景**：
- 开发测试
- 演示 Demo
- 无外网环境

### LLM 模式

`USE_MOCK_LLM=false` 时，系统调用 OpenAI API 生成 SQL：

- 优点：理解自然语言，适应性强
- 缺点：需要 API Key，有调用成本

**适用场景**：
- 生产环境
- 复杂问题分析
- 需要语义理解

---

## 安全建议

1. **禁止提交 .env 文件**到 Git 仓库（已在 .gitignore 中配置）
2. **生产环境**使用密钥管理服务（如 AWS Secrets Manager）
3. **API Key** 定期轮换
4. **数据库密码**使用强密码

## 故障排查

### PostgreSQL 连接失败

```
RuntimeError: 缺少环境变量：POSTGRES_DSN
```

**解决方案**：
1. 检查 .env 文件是否存在
2. 检查 POSTGRES_DSN 格式是否正确
3. 验证 PostgreSQL 服务是否启动

### MySQL 连接失败

```
RuntimeError: 缺少环境变量：MYSQL_HOST
```

**解决方案**：
1. 检查 MySQL 服务是否启动
2. 验证用户名/密码是否正确
3. 确认数据库 `nl2sql_platform` 已创建

### LLM 调用失败

```
openai.AuthenticationError: Incorrect API key provided
```

**解决方案**：
1. 检查 `OPENAI_API_KEY` 是否正确
2. 如使用代理，检查 `OPENAI_API_BASE` 配置
3. 临时切换到 Mock 模式：`USE_MOCK_LLM=true`
