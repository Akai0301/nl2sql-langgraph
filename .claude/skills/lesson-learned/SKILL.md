# Lesson-Learned Skill — 错误记录与经验沉淀

## 触发条件

此 Skill 由以下两种方式激活：
1. **Hook 自动激活**：用户消息包含纠正关键词（不对/错了/应该是/记住/下次...）
2. **显式命令**：用户执行 `/remember`

---

## 核心职责

**三步流程**：记录 → 分类 → 沉淀

---

## 第一步：识别与确认

收到用户纠正时，先用一句话确认理解：

```
我理解到：你纠正了 [具体错误行为]，正确做法是 [正确行为]。
是否要将此经验记录到项目经验库？
```

---

## 第二步：分类错误

按以下 6 个类别分类，决定记录位置和严重程度：

| 类别代码 | 说明 | 严重度 | 是否同步 CLAUDE.md |
|---------|------|--------|-------------------|
| `LANG` | LangGraph 架构违规（节点返回非 dict、State 定义错误） | 🔴 强制 | ✅ 必须同步 |
| `CODE` | 代码规范违反（命名/禁用写法/对象转换） | 🟠 重要 | ✅ 通常同步 |
| `TOOL` | 工具/API 误用（psycopg/PyMySQL/OpenAI SDK） | 🟡 建议 | 视频率决定 |
| `SKILL` | Skill 理解有误（触发条件/适用范围） | 🟡 建议 | 更新对应 SKILL.md |
| `LOGIC` | 业务逻辑错误（特定场景的处理方式） | 🟠 重要 | 视情况 |
| `OTHER` | 其他经验（用户偏好/项目特有约定） | 🟢 参考 | 一般不同步 |

---

## 第三步：写入经验库

### 3.1 写入 `.claude/memory/lessons.md`

在文件的 `<!-- LOG_START -->` 和 `<!-- LOG_END -->` 之间，**顶部追加**以下格式：

```markdown
### [RXXX] YYYY-MM-DD | 类别代码 | 严重度

**❌ 错误行为**：Agent 做了什么

**✅ 正确做法**：应该怎么做

**根本原因**：为什么会犯这个错

**沉淀规则**：用一句话总结，下次直接查此规则

**影响范围**：哪些 Skill / 哪类任务需要注意

---
```

### 3.2 更新规则速查表

在 `<!-- RULES_START -->` 和 `<!-- RULES_END -->` 之间，追加一行：

```markdown
| [RXXX] | 类别 | 一句话规则摘要 |
```

### 3.3 判断是否同步到 CLAUDE.md

**必须同步**（类别为 `LANG` 或 `CODE`，且严重度为强制/重要）：
- 在 `CLAUDE.md` 的 `## 绝对禁止的写法` 区域追加对应禁止项
- 格式：`# ❌ 禁止X: [一句话说明]`

**选择性同步**（类别为 `SKILL`）：
- 找到对应的 `.claude/skills/xxx/SKILL.md`
- 在适当位置补充注意事项或纠正说明

---

## 第四步：记录后的反馈

完成记录后，回复用户：

```
✅ 已记录到项目经验库：

**[RXXX]** 类别: LANG | 严重度: 🔴 强制

规则：[一句话规则摘要]

[是否同步到了 CLAUDE.md / 对应 SKILL.md]

建议提交：git add .claude/memory/lessons.md && git commit -m "docs(.claude): 记录经验 RXXX - [简短描述]"
```

---

## ID 编号规则

- 格式：`R` + 3位数字，从当前文件最大编号 +1
- 示例：现有 R001、R002，下一个是 R003
- 如果文件为空（首次记录），从 R001 开始

---

## 重要约束

1. **不要过度记录**：只记录真实发生的纠正，不要猜测
2. **简洁胜于全面**：规则摘要必须 ≤30 字，能一眼看懂
3. **及时性**：在用户确认后立刻写入，不要等到 Stop Hook
4. **类别准确**：错误分类直接影响是否同步到 CLAUDE.md

## 参考：典型记录示例

### LangGraph 架构违规

```markdown
### [R001] 2026-04-02 | LANG | 🔴 强制

**❌ 错误行为**：在节点函数中直接返回了字符串，而不是字典

**✅ 正确做法**：节点函数必须返回 dict，返回值会与 State 合并

**根本原因**：误解 LangGraph 节点函数的返回机制

**沉淀规则**：节点函数返回 dict | LangGraph 节点必须返回字典

**影响范围**：langgraph-flow, 所有节点开发任务

---
```

### 数据库工具误用

```markdown
### [R002] 2026-04-02 | TOOL | 🟡 建议

**❌ 错误行为**：在 MySQL 操作中使用了 psycopg 的参数占位符 `%s`

**✅ 正确做法**：MySQL 使用 PyMySQL，参数占位符也是 `%s`，但连接方式不同

**根本原因**：混淆了 PostgreSQL 和 MySQL 的驱动

**沉淀规则**：区分 psycopg 和 PyMySQL | PostgreSQL 和 MySQL 使用不同连接函数

**影响范围**：database-ops, db-meta-query

---
```

### SQL 生成错误

```markdown
### [R003] 2026-04-02 | LOGIC | 🟠 重要

**❌ 错误行为**：生成的 SQL 使用了不存在的字段名 `order_amount`

**✅ 正确做法**：根据 lake_table_metadata 表中的 measure_column 字段确定正确的列名

**根本原因**：未正确使用元数据检索结果

**沉淀规则**：字段名从元数据获取 | SQL 字段名必须来自元数据检索

**影响范围**：nl2sql-pipeline, sql_generation_node

---
```

### SSE 流式接口错误

```markdown
### [R004] 2026-04-02 | CODE | 🟠 重要

**❌ 错误行为**：SSE 接口返回了 JSON 格式而不是 `data: ...\n\n` 格式

**✅ 正确做法**：SSE 响应必须使用 `data: {json}\n\n` 格式，每条消息以两个换行结束

**根本原因**：对 SSE 协议理解不足

**沉淀规则**：SSE 格式 `data: json\n\n` | SSE 响应必须符合协议格式

**影响范围**：websocket-sse, streaming.py

---
```
