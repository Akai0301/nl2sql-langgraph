## 技术选型决策树

### 场景 1：需要认证授权？

```
需要认证授权？
├─ 是 → JWT (PyJWT)
│      适用：API 认证、用户登录、权限控制
│
└─ 否 → 无需认证
```

### 场景 2：需要数据库操作？

```
需要数据库操作？
├─ 是 → SQLAlchemy ORM
│      适用：CRUD 操作、事务管理、关系映射
│
└─ 否 → 不需要数据库
```

### 场景 3：需要缓存？

```
需要缓存？
├─ 是 → Redis
│      适用：会话存储、缓存数据、分布式锁
│
└─ 否 → 不需要缓存
```

### 场景 4：需要定时任务？

```
需要定时任务？
├─ 是 → APScheduler
│      适用：周期性任务、一次性任务、延迟任务
│
└─ 否 → 不需要定时任务
```

### 场景 5：需要数据库迁移？

```
需要数据库迁移？
├─ 是 → Alembic
│      适用：数据库结构变更、版本管理
│
└─ 否 → 不需要数据库迁移
```

## 常见选型对比

### 1. Web 框架选型

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **FastAPI** | 现代 API 开发 | 高性能、自动文档、类型提示 | 相对较新 |
| **Flask** | 轻量应用 | 灵活、生态丰富 | 性能一般、需要手动集成 |
| **Django** | 全栈应用 | 功能全面、内置管理后台 | 重量级、学习曲线陡 |

**本项目推荐**：FastAPI

### 2. ORM 框架选型

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **SQLAlchemy** | 复杂数据库操作 | 功能全面、异步支持、成熟稳定 | 学习曲线较陡 |
| **Peewee** | 简单应用 | 轻量、易用 | 功能相对简单 |
| **Tortoise ORM** | 异步应用 | 原生异步、语法简洁 | 生态相对较小 |

**本项目推荐**：SQLAlchemy

### 3. 缓存选型

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **Redis** | 高性能缓存 | 丰富的数据结构、持久化支持 | 需要额外部署 |
| **Memcached** | 简单缓存 | 轻量、高性能 | 功能相对简单 |
| **本地缓存** | 简单场景 | 无额外依赖 | 不支持分布式 |

**本项目推荐**：Redis

### 4. 定时任务选型

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **APScheduler** | 通用定时任务 | 简单易用、功能丰富、支持多种触发器 | 不支持分布式调度 |
| **Celery** | 复杂任务队列 | 分布式支持、结果存储、重试机制 | 配置复杂、需要额外依赖 |
| **自定义定时任务** | 简单场景 | 轻量、可控 | 功能有限、需要自己实现 |

**本项目推荐**：APScheduler

### 5. 数据库迁移选型

| 方案 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **Alembic** | SQLAlchemy 项目 | 官方推荐、与 SQLAlchemy 集成紧密 | 学习曲线较陡 |
| **SQLModel** | Pydantic 模型 | 与 Pydantic 集成、语法简洁 | 相对较新 |
| **自定义迁移脚本** | 简单场景 | 灵活、可控 | 维护成本高 |

**本项目推荐**：Alembic

## 评估检查清单

### 选型前必查

- [ ] **功能满足**：是否满足当前功能需求？
- [ ] **性能满足**：是否满足性能要求？
- [ ] **社区活跃**：GitHub Stars、Issues 响应速度？
- [ ] **文档完善**：是否有中文文档？
- [ ] **兼容性**：与现有技术栈是否兼容？
- [ ] **学习成本**：团队是否熟悉？学习曲线如何？
- [ ] **维护成本**：长期维护成本如何？
- [ ] **安全漏洞**：是否有已知安全漏洞？
- [ ] **License**：是否允许商用？
- [ ] **项目已有**：项目是否已经集成？

### 本项目优先原则

1. **优先使用项目已集成的技术**：已测试、风格统一
2. **优先使用 Python 标准库**：稳定、无需额外依赖
3. **优先使用主流技术**：社区活跃、文档完善
4. **避免重复造轮子**：先查项目是否有现成实现
