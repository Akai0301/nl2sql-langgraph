## 2. 技术需求

### 2.1 后端技术栈
- Python: 3.12+
- FastAPI: 最新版
- LangGraph: 流程编排
- LangChain: LLM 集成
- OpenAI SDK: 模型调用
- psycopg: PostgreSQL 驱动
- PyMySQL: MySQL 驱动
- Pydantic: 数据验证

### 2.2 前端技术栈
- Vue: 3.x
- TypeScript: 最新版
- Pinia: 状态管理
- Element Plus: UI 组件库
- Vue Flow: 流程图可视化
- ECharts: 图表展示
- TailwindCSS: 样式框架

### 2.3 数据库
- PostgreSQL: 业务数据（知识库、指标、事实表）
- MySQL: 平台数据（历史记录）

### 2.4 架构规范
- LangGraph 流程：State（TypedDict）+ Node（函数）+ Graph（StateGraph）
- 后端无三层架构，采用模块化设计（nodes.py、tools.py、graph_builder.py）
- 前端采用 Pinia Store 管理查询状态

## 3. 非功能性需求

### 3.1 性能要求
- SQL 执行时间 < 5s
- SSE 流式响应延迟 < 100ms
- 并行检索节点响应时间 < 2s

### 3.2 安全要求
- SQL 只允许 SELECT 语句（禁止 INSERT/UPDATE/DELETE）
- API Key 不硬编码，使用环境变量
- 数据库连接串加密存储

### 3.3 可维护性
- 节点函数返回 dict（与 State 合并）
- State 定义使用 TypedDict
- 工具函数封装在 tools.py

## 4. 验收标准
- 功能完整性: 核心流程 100%
- SQL 生成准确率: Mock 模式 100%，LLM 模式 > 80%
- 性能达标: SQL 执行 < 5s
- 文档完整: CLAUDE.md + project.md + 表结构文档
