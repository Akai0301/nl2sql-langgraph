## 🎯 执行流程

### 第一步：快速扫描项目状态

```bash

git log -5 --oneline

# 扫描所有代码中的 TODO/FIXME
Grep pattern: "TODO:|FIXME:" path: app/ glob: "*.py" output_mode: content -B 2

# 检查 LangGraph 代码规范
```

### 第二步：分析开发阶段

根据代码完整性和功能数量判断：

| 阶段 | 特征 | 功能完成率 |
|------|------|----------|
| 🟢 初期 | 基础框架搭建，节点函数较少 | < 30% |
| 🟡 中期 | 核心节点开发中，流程逐步完善 | 30-70% |
| 🔴 后期 | 流程基本完成，处于优化阶段 | > 70% |

### 第三步：分类问题和建议

- **紧急问题**：SQL 执行失败、节点异常、SSE 连接错误
- **重要问题**：节点逻辑缺陷、状态定义不完整、工具函数异常处理
- **优化建议**：查询性能优化、前端体验提升、文档完善

## 🎓 开发建议

### 开发流程建议

1. **明确需求**：使用 `/start` 快速了解项目结构
2. **检查规范**：使用 `/check` 检查代码质量
3. **分析进度**：使用 `/progress` 了解完成情况
4. **开始开发**：参考 app/nodes.py 开发新节点
5. **添加待办**：使用 `/add-todo` 跟踪任务
6. **获取建议**：使用 `/next` 确定下一步方向

### 最佳实践

| 实践 | 说明 |
|------|------|
| State TypedDict | 使用 TypedDict 定义 State，total=False |
| 节点返回 dict | 每个节点函数必须返回字典 |
| async/await | 所有节点和工具函数使用异步 |
| 异常处理 | 节点内 try-except，返回 error 字段 |
| 前端 TypeScript | 必须有类型定义 |
| 定期检查 | 每周至少一次运行 `/check` 和 `/progress` |

### 技术选型建议

| 场景 | 推荐方案 | 理由 |
|------|--------|------|
| 开发新节点 | 参考 app/nodes.py | 遵循 LangGraph 规范 |
| 开发前端组件 | Vue 3 + TailwindCSS | 项目标准技术栈 |
| 代码规范检查 | `/check` 命令 | 及时发现问题 |
| 项目进度追踪 | `/progress` 命令 | 定期了解完成情况 |
