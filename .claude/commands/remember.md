# /remember — 显式记录经验

## 功能

将当前对话中的重要经验、用户偏好、项目约定主动记录到项目经验库。

## 使用方式

```
/remember [可选：要记录的内容]
```

- 不带参数：Agent 回顾当前对话，提取值得记录的经验并询问确认
- 带参数：直接记录用户提供的内容

## 执行流程

1. **回顾当前对话**，找出以下类型的内容：
   - 用户的纠正（明确的不对/错了/应该是）
   - 用户表达的偏好（我更喜欢/以后都用）
   - 项目特有约定（这个项目里/我们规定）
   - 你自己发现的规律（通常X情况下应该Y）

2. **列出候选项**，让用户确认哪些要记录：
   ```
   发现以下值得记录的内容：

   1. [描述] — 类别: ARCH | 你要记录这条吗？(y/n)
   2. [描述] — 类别: OTHER | 你要记录这条吗？(y/n)
   ```

3. **对确认的条目**，激活 `lesson-learned` Skill 执行记录

## 注意

- 此命令不跳过 Skill 评估流程
- 记录完成后，给出 git commit 建议
